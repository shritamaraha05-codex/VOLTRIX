05 — Database Setup (Supabase)

Create the PostgreSQL schema

1

Create a Supabase project at supabase.com. Choose a region close to your users. Note your project password.

2

Get your connection string. Go to Project Settings → Database → Connection string → URI. Copy the full URI.

postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-REF].supabase.co:5432/postgres
3

Run the schema. Go to the SQL Editor in Supabase dashboard, paste the contents of backend/schema.sql, and click "Run".

This creates 6 tables: zones, households, forecasts, stress_events, recommendations, simulation_state — plus indexes and seed data for 5 zones.

4

Paste the connection string into your .env as DATABASE_URL.

Tables Created

zonesGrid zones (wards)bq_zone_id, name, baseline_capacity_kw, household_count
householdsIndividual householdsbq_household_id, zone_id (FK), archetype, email
forecastsHourly load predictionszone_id (FK), forecast_for, predicted_load_kw, actual_load_kw
stress_eventsDetected grid stresszone_id (FK), severity, predicted_peak_kw, capacity_kw, reasoning
recommendationsAI-generated nudges/alertsstress_event_id (FK), target_type, message, sent
simulation_stateDay counter (singleton)current_day (1-30)

06 — BigQuery Setup (GCP)

Load smart-meter data into Google BigQuery

1

Create a GCP project at console.cloud.google.com. Note the Project ID.

2

Enable BigQuery API in the GCP Console → APIs & Services → Enable APIs → search "BigQuery API" → Enable.

3

Create a service account with roles: BigQuery Data Editor, BigQuery Job User. Download the JSON key and save as backend/service-account.json.

4

Set GCP_PROJECT in your .env to your project ID.

5

Load the CSV data into BigQuery using the provided script:

cd backend
pip install -r requirements.txt
python ../load_to_bq.py

This creates dataset voltrix and 3 tables:

  • voltrix.load_readings — 43,200 hourly readings (clustered by zone_id, timestamp)
  • voltrix.zone_capacity — 5 zone metadata rows
  • voltrix.households — 60 household records

BigQuery Tables

TableColumnsRows

voltrix.zone_capacityzone_id, zone_name, baseline_capacity_kw, household_count5

For Cloud Run production: The backend uses the Cloud Run service account identity for BigQuery access. You do NOT need GOOGLE_APPLICATION_CREDENTIALS in production — just ensure the Cloud Run service account has BigQuery roles.

07 — Gemini / ADK Setup

AI agent for stress analysis and chat

1

Get a Gemini API Key from Google AI Studio → Create API Key.

2

Set GEMINI_API_KEY in your .env. The backend also sets GOOGLE_API_KEY automatically (required by Google ADK).

What the AI Does

Agent: voltrix_grid_intelligence_agent

Model: gemma-4-31b-it (Google Gemma 4 31B Instruct) via AI Studio

3 Tools:

  • get_zone_details — Calculates overage %, risk level (critical/moderate) from predicted peak vs capacity
  • analyse_stress_cause — Determines root causes (evening peak, WFH load, family appliances, etc.)
  • generate_household_nudges — Creates personalized energy-saving messages per household archetype

Chat: Uses google-genai SDK directly (no ADK) for SSE streaming. Token-by-token response via Server-Sent Events.

Free tier: Gemini API has a generous free tier. Check pricing for current limits.

09 — Backend — Local Development

Run the FastAPI server locally

1

Clone the repo and navigate to backend:

git clone https://github.com/shritamaraha05-codex/VOLTRIX.git
cd VOLTRIX/backend
2

Create a virtual environment and install dependencies:

python -m venv .venv

# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
3

Configure environment variables:

cp .env.example .env
# Edit .env with your credentials (see Section 04)
4

Run the server:

# Option A — use the launcher script:
bash run.sh

# Option B — manual:
set -a && source .env && set +a
uvicorn main:app --reload --port 8000 --log-level info
5

Verify: Open http://localhost:8000/docs — you should see the Swagger UI with all endpoints.

curl http://localhost:8000/health
# → {"status":"ok","service":"voltrix-api","day":1}

10 — Backend — Cloud Run Deploy

Deploy to Google Cloud Run

1

Install and authenticate gcloud CLI:

gcloud auth login
gcloud config set project YOUR_PROJECT_ID
2

Enable required GCP APIs:

gcloud services enable run.googleapis.com
gcloud services enable containerregistry.googleapis.com
gcloud services enable bigquery.googleapis.com
3

Set environment variables in your shell (or edit deploy.sh):

export PROJECT_ID="your-gcp-project-id"
export DATABASE_URL="postgresql://postgres:PASSWORD@db.REF.supabase.co:5432/postgres"
export GEMINI_API_KEY="AIzaSy..."
export SMTP_HOST="smtp.gmail.com"
export SMTP_PORT="587"
export SMTP_USER="your-email@gmail.com"
export SMTP_PASSWORD="your-app-password"
export SMTP_FROM="VOLTRIX Alerts <your-email@gmail.com>"
export VOLTRIX_OPS_EMAIL="ops@yourcompany.com"
export ALLOWED_ORIGINS="https://your-app.vercel.app,https://your-app.pages.dev"
4

Deploy with one command (edit backend/deploy.sh first with your project ID):

cd backend
bash deploy.sh

This builds the Docker image, pushes to GCR, and deploys to Cloud Run. Takes 2-5 minutes.

Dockerfile Breakdown

FROM python:3.11-slim
RUN apt-get update && apt-get install -y --no-install-recommends gcc libpq-dev
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV PORT=8080
EXPOSE 8080
CMD ["sh", "-c", "uvicorn main:app --host 0.0.0.0 --port ${PORT}"]

12 — Frontend — Local Development (Vite)

Run the dashboard with hot reload

1

Navigate to frontend and install dependencies:

cd frontend
npm install
2

Configure backend URL. Edit config.js:

// For local dev with Vite proxy — leave empty:
window.__BACKEND_URL = "";

// For pointing to a deployed backend:
window.__BACKEND_URL = "https://voltrix-backend-xxxxx-uc.a.run.app";
3

Start the dev server:

npm run dev

Opens at http://localhost:5173. Vite proxies all API calls to localhost:8000.

Prerequisite: The backend must be running on localhost:8000 for the Vite proxy to work. Start the backend first (Section 09).

Vite Proxy Routes

All these paths are proxied from port 5173 → port 8000:

/health, /zones, /stress-events, /recommendations,
/simulation, /simulate, /chat, /ingest, /admin

Build and deploy the dashboard

Build

cd frontend
npm run build
# Output: frontend/dist/index.html (single file, ~55KB)

Deploy to Vercel (Recommended)

1

Edit config.js with your deployed backend URL:

window.__BACKEND_URL = "https://voltrix-backend-xxxxx-uc.a.run.app";
2

Rebuild and deploy:

npm run build
npx vercel --prod
3

Update CORS: Add your Vercel URL to ALLOWED_ORIGINS in the backend .env and redeploy.

Deploy to Netlify

npm run build
npx netlify deploy --prod --dir=dist

Deploy to Any Static Host

The build output is a single dist/index.html file. Upload the dist/ folder to any static host (GitHub Pages, Cloudflare Pages, Nginx, etc.).

Important: After deploying, add your frontend URL to ALLOWED_ORIGINS in the backend .env or Cloud Run env vars, otherwise API calls will be blocked by CORS.

15 — Troubleshooting

Common issues and fixes

psycopg2-binary fails to install

Version 2.9.9 has no pre-built wheel for Python 3.14. Fix: unpin in requirements.txt:

psycopg2-binary>=2.9.9
pandas fails to install (no pg_config / vswhere.exe)

Pinned version has no wheel for your Python. Fix: unpin:

pandas>=2.2.2
numpy>=1.26.4
CORS errors in browser console

Your frontend URL is not in ALLOWED_ORIGINS. Add it (including http:// prefix, and null for local file:// testing).

Backend shows "Offline" in dashboard

Check: (1) Backend is running on port 8000, (2) config.js has correct URL or empty for Vite proxy, (3) Browser console for CORS errors.

DATABASE_URL connection refused

Verify: (1) Supabase project is active (not paused), (2) Password is correct, (3) Your IP is in Supabase allowlist (Settings → Database → Network restrictions).

Gemini API errors

Check: (1) API key is valid, (2) Gemini API is enabled in GCP, (3) You haven't exceeded free tier limits. The backend has fallback responses if Gemini fails.

Email not sending

Check: (1) Using Gmail App Password (not account password), (2) SMTP_PORT matches (587 for STARTTLS, 465 for SSL), (3) 2FA is enabled on Gmail account.
